# Sequence Diagram — Luồng tiếp khách (Tenant Onboarding)

Tài liệu mô tả **sequence diagram đầy đủ** cho luồng onboarding khách thuê trong SLMS2026, đã chỉnh khớp **code BE hiện tại** (tháng 7/2026).

**Tham chiếu code:** `TenantOnboardingServiceImpl`, `TenantContractController`, `TenantContractActionController`, `TenantContractDocumentServiceImpl`, `PayosWebhookController`, `OcrController`

**Tài liệu liên quan:** [`FE-BE-tenant-onboarding-flow.md`](./FE-BE-tenant-onboarding-flow.md) · [`tenant-contract-template-spec.md`](./tenant-contract-template-spec.md)

---

## 1. Tổng quan

Luồng mobile tiếp khách (happy path) gồm **3 giai đoạn**:

| Giai đoạn | Vai trò chính | Trạng thái HĐ | Việc chính |
|-----------|---------------|---------------|------------|
| **1 — Chuẩn bị HĐ nháp** | Admin | `DRAFT` | Chọn BĐS/phòng, OCR/nhập form, tạo nháp, gán Manager |
| **2 — Tiếp khách & thu cọc** | Manager | `DRAFT` → `PENDING` → `PAID` | Mở HĐ, tạo link PayOS, khách thanh toán, xuất DOCX nháp |
| **3 — Kích hoạt HĐ** | Manager + Khách | `ACTIVE` | Gửi OTP SMS, xác nhận, tạo tài khoản tenant, phòng `RENTED` |

### Thuật ngữ trạng thái

| Khái niệm nghiệp vụ | Trong hệ thống |
|---------------------|----------------|
| Hợp đồng **nháp** | `ContractStatus.DRAFT` — chưa tạo `User`/`Tenant` |
| Hợp đồng **chờ tiếp khách** | `ContractStatus.PENDING` + `paymentStatus` null hoặc `PENDING` |
| Đã thanh toán cọc | `PaymentStatus.PAID` |
| Hợp đồng **active** | `ContractStatus.ACTIVE` |

### State machine (rút gọn)

```mermaid
stateDiagram-v2
    [*] --> DRAFT: POST tenant-contract<br/>body draft=true
    DRAFT --> PENDING: POST deposit-payment
    PENDING --> PENDING: paymentStatus=PENDING<br/>(tạo link PayOS)
    PENDING --> PENDING: paymentStatus=PAID<br/>(webhook / check-payment)
    PENDING --> ACTIVE: confirm + OTP hợp lệ

    DRAFT --> TERMINATED: cancel
    PENDING --> TERMINATED: cancel
    ACTIVE --> EXPIRED: quá endDate
    ACTIVE --> TERMINATED: checkout / hủy
```

## 2. Participants (thành phần)

| Participant | Vai trò | Ghi chú |
|-------------|---------|---------|
| **Admin** | Nhập liệu, tạo HĐ nháp, gán Manager | `ROLE_ADMIN` |
| **Manager** | Tiếp khách tại chỗ, thu cọc, OTP | `ROLE_MANAGER` |
| **Khách** | Quét QR PayOS, đọc OTP SMS | Không gọi API trực tiếp |
| **pdfExtract.ts** | Utility **FE** — trích xuất field từ file PDF HĐ cũ | Client-side, **không** phải BE |
| **OcrService** | BE — đọc chỉ số đồng hồ / hóa đơn EVN từ ảnh | `POST /api/v1/ocr/meter`, `/ocr/evn-bill` |
| **TenantContractController** | REST — tạo HĐ theo property/room | `POST .../tenant-contract` |
| **TenantContractActionController** | REST — thao tác theo `contractId` | deposit, OTP, assign-manager, … |
| **TenantOnboardingServiceImpl** | Service chính onboarding | `onboardTenant`, `assignManager`, PayOS, OTP, confirm |
| **TenantContractDocumentServiceImpl** | Fill template DOCX | `autoGenerateDocument` sau `PAID` và `confirm` |
| **TenantContractRepository** | Persistence `TenantContract` | |
| **PayOS** | Cổng thanh toán | Webhook HMAC |
| **OtpService** | Gửi/xác minh OTP SMS (Twilio) | |
| **PayosWebhookController** | Nhận webhook thanh toán | Public, không JWT |

> **Lưu ý:** Không có `TenantContractService` hay `createDraftContract()` riêng — toàn bộ logic nằm trong `TenantOnboardingServiceImpl.onboardTenant()` với `draft: true` trong body.

## 3. Giai đoạn 1 — Admin chuẩn bị HĐ nháp (`DRAFT`)

Bao gồm: chọn BĐS/phòng, tra khách, OCR (tuỳ chọn), tạo nháp, chỉnh sửa nháp, gán Manager.

```mermaid
sequenceDiagram
    autonumber
    actor Admin
    participant FE as FE (Web Admin)
    participant PdfExtract as pdfExtract.ts (FE)
    participant Ocr as OcrController / OcrService
    participant PropCtrl as PropertyController / RoomController
    participant LookupCtrl as TenantLookupController
    participant ContractCtrl as TenantContractController
    participant ActionCtrl as TenantContractActionController
    participant Onboard as TenantOnboardingServiceImpl
    participant Repo as TenantContractRepository
    participant DB as Database

  Note over Admin,DB: A. Chọn BĐS & phòng

    Admin->>FE: Mở màn tiếp khách
    FE->>PropCtrl: GET /api/v1/properties/rentable
    PropCtrl->>DB: danh sách BĐS cho thuê
    PropCtrl-->>FE: Property[]

    Admin->>FE: Chọn property (+ phòng nếu thuê phòng)
    FE->>PropCtrl: GET /api/v1/properties/{id}
    FE->>PropCtrl: GET /api/v1/properties/{id}/rooms
    PropCtrl-->>FE: Property, Room[]

  Note over Admin,DB: B. Tra cứu khách theo SĐT

    Admin->>FE: Nhập SĐT (blur)
    FE->>LookupCtrl: GET /api/v1/tenants/lookup?phone=...
    LookupCtrl->>DB: tìm User theo SĐT
    LookupCtrl-->>FE: { exists, fullName, cccd, role }
    Note right of FE: exists=true → tự điền form<br/>role ∉ USER/TENANT → cảnh báo FE

  Note over Admin,DB: C. OCR / nhập liệu (tuỳ chọn)

    alt Upload PDF hợp đồng cũ
        Admin->>FE: Upload file PDF
        FE->>PdfExtract: extractContractFields(pdf)
        PdfExtract-->>FE: fullName, cccd, phone, dates, …
        Note right of PdfExtract: Chỉ chạy trên FE — không gọi BE
    end

    opt Ảnh đồng hồ điện/nước
        Admin->>FE: Upload ảnh đồng hồ
        FE->>Ocr: POST /api/v1/ocr/meter { imageUrl }
        Ocr-->>FE: initialElectricReading / initialWaterReading
    end

    Admin->>FE: Hoàn tất form (giá, cọc, ngày, ảnh bàn giao, …)

  Note over Admin,DB: D. Tạo HĐ nháp

    FE->>ContractCtrl: POST /api/v1/properties/{id}/rooms/{roomId}/tenant-contract<br/>body: { draft: true, fullName, phoneNumber, cccd, … }
    ContractCtrl->>Onboard: onboardTenant(propertyId, roomId, request)

    Onboard->>Onboard: validate ngày, phòng trống, trùng HĐ
    Note right of Onboard: draft=true → bỏ qua validate moveInDate=hôm nay

    Onboard->>Onboard: skip getOrCreateTenant()
    Note right of Onboard: Lưu draftTenantName/Phone/Cccd<br/>tenant = null

    Onboard->>Repo: save(TenantContract status=DRAFT)
    Repo->>DB: INSERT
    Repo-->>Onboard: TenantContract (DRAFT)
    Onboard-->>ContractCtrl: TenantContractResponse
    ContractCtrl-->>FE: 201 Created { id, contractCode, status: DRAFT }
    FE-->>Admin: Hiển thị HĐ nháp + contractId

  Note over Admin,DB: E. Chỉnh sửa nháp (tuỳ chọn)

    opt Admin sửa thông tin sau OCR
        Admin->>FE: Cập nhật form
        FE->>ActionCtrl: PUT /api/v1/tenant-contracts/{id}
        ActionCtrl->>Onboard: updateDraftContract(id, request)
        Onboard->>Repo: update (chỉ khi status=DRAFT)
        Repo-->>Onboard: TenantContract (DRAFT)
        Onboard-->>ActionCtrl: TenantContractResponse
        ActionCtrl-->>FE: 200 OK
    end

  Note over Admin,DB: F. Gán Manager đón khách

    alt Gán ngay khi tạo (cùng POST body)
        Note over FE,Onboard: assignedManagerId + expectedReceptionDate<br/>đã set lúc bước D
    else Gán sau (bước riêng)
        Admin->>FE: Chọn Manager + ngày tiếp khách dự kiến
        FE->>ActionCtrl: PATCH /api/v1/tenant-contracts/{id}/assign-manager<br/>{ assignedManagerId, expectedReceptionDate }
        ActionCtrl->>Onboard: assignManager(id, request)
        Onboard->>Repo: update(assignedManager, expectedReceptionDate)
        Repo-->>Onboard: TenantContract (DRAFT)
        Onboard-->>ActionCtrl: TenantContractResponse
        ActionCtrl-->>FE: 200 OK
    end

    Note over Onboard: 📋 Planned: push notification tới Manager<br/>— chưa implement trong assignManager()

    FE-->>Admin: Nút "Tiếp khách" → deep link contractId cho Manager
```

## 4. Giai đoạn 2 — Manager tiếp khách & thu cọc (`DRAFT` → `PENDING` → `PAID`)

```mermaid
sequenceDiagram
    autonumber
    actor Manager
    actor Khach as Khách thuê
    participant FE as FE (Mobile Manager)
    participant ActionCtrl as TenantContractActionController
    participant Onboard as TenantOnboardingServiceImpl
    participant Doc as TenantContractDocumentServiceImpl
    participant Repo as TenantContractRepository
    participant PayOS as PayOS
    participant Webhook as PayosWebhookController
    participant DB as Database

    Note over Manager,DB: A. Mở HĐ được gán

    Manager->>FE: Mở app / deep link contractId

    alt Deep link trực tiếp
        FE->>ActionCtrl: GET /api/v1/tenant-contracts/{id}
    else Danh sách HĐ được gán
        FE->>ActionCtrl: GET /api/v1/tenant-contracts/managed?status=DRAFT
        Note right of ActionCtrl: Lọc theo assignedManager = user hiện tại
    end

    ActionCtrl->>Onboard: getContract(id) / getManagedContracts(status)
    Onboard->>Repo: findById / findManagedContractsByStatus
    Repo-->>Onboard: TenantContract (DRAFT)
    Onboard-->>FE: TenantContractResponse
    FE-->>Manager: Chi tiết HĐ — preview JSON, chưa có file Word

    Note over Manager,DB: B. Kiểm tra duyệt giá Host (nếu có)

    opt requireHostPriceApproval = true
        Note over FE: Chờ priceApprovalStatus = APPROVED_AWAITING_DEPOSIT<br/>FE disable nút Thu cọc khi PENDING_PRICE_APPROVAL
    end

    Note over Manager,DB: C. Tạo link thanh toán cọc

    Manager->>FE: Bấm "Thu cọc"
    FE->>ActionCtrl: POST /api/v1/tenant-contracts/{id}/deposit-payment
    ActionCtrl->>Onboard: createDepositPayment(id)

    Onboard->>Onboard: DRAFT → PENDING (validate moveInDate, phòng trống)
    Onboard->>PayOS: createPaymentLink(orderCode, deposit)
    PayOS-->>Onboard: checkoutUrl + qrCode
    Onboard->>Repo: save(payosOrderCode, paymentStatus=PENDING)
    Onboard-->>ActionCtrl: TenantContractResponse + payosCheckoutUrl + payosQrCode
    ActionCtrl-->>FE: 200 OK
    FE-->>Manager: Hiển thị QR / link PayOS
    FE-->>Khach: Quét QR thanh toán

    Note over Manager,DB: D. Xác nhận thanh toán

    alt Production — Webhook tự động
        PayOS->>Webhook: POST /api/v1/payos/webhook (HMAC)
        Webhook->>Onboard: markDepositPaid(orderCode)
        Onboard->>Repo: paymentStatus=PAID, paidAt=now
        Onboard->>Doc: autoGenerateDocument(contractId)
        Doc->>DB: lưu documentUrl (DOCX nháp)
    else Dev / fallback — Manager bấm kiểm tra
        Manager->>FE: Bấm "Kiểm tra thanh toán"
        FE->>ActionCtrl: POST /api/v1/tenant-contracts/{id}/check-payment
        ActionCtrl->>Onboard: syncPaymentStatus(id)
        Onboard->>PayOS: getPaymentStatus(orderCode)
        Onboard->>Doc: autoGenerateDocument() nếu PAID
    else Poll trạng thái
        FE->>ActionCtrl: GET /api/v1/tenant-contracts/{id}
        Note right of FE: Lặp đến khi paymentStatus=PAID
    end

    Onboard-->>FE: TenantContractResponse (paymentStatus=PAID, documentUrl)
    FE-->>Manager: Tải/xem DOCX nháp — GET /tenant-contracts/{id}/document
```

## 5. Giai đoạn 3 — Xác nhận OTP & kích hoạt HĐ (`ACTIVE`)

```mermaid
sequenceDiagram
    autonumber
    actor Manager
    actor Khach as Khách thuê
    participant FE as FE (Mobile Manager)
    participant ActionCtrl as TenantContractActionController
    participant Onboard as TenantOnboardingServiceImpl
    participant Otp as OtpService (Twilio)
    participant Doc as TenantContractDocumentServiceImpl
    participant Repo as TenantContractRepository
    participant UserRepo as UserRepository
    participant DB as Database

    Note over Manager,DB: Điều kiện: paymentStatus = PAID

    Manager->>FE: Bấm "Gửi OTP xác nhận"
    FE->>ActionCtrl: POST /api/v1/tenant-contracts/{id}/send-otp
    ActionCtrl->>Onboard: sendContractConfirmOtp(id)
    Onboard->>Otp: sendOtp(phone, CONTRACT_CONFIRM, contractId)
    Otp-->>Khach: SMS mã OTP
    Note right of Otp: Dev mode: không gửi SMS thật

    Khach-->>Manager: Đọc mã OTP
    Manager->>FE: Nhập OTP
    FE->>ActionCtrl: POST /api/v1/tenant-contracts/{id}/confirm { otp }
    ActionCtrl->>Onboard: confirmContract(id, otp)

    Onboard->>Otp: verifyOrThrow(phone, otp)
    Otp-->>Onboard: OK

    alt tenant = null (luồng DRAFT)
        Onboard->>Onboard: getOrCreateTenant(draftPhone, draftName, draftCccd)
        Onboard->>UserRepo: tạo User t{phone} hoặc promote ROLE_USER → TENANT
        UserRepo->>DB: INSERT/UPDATE User + Tenant
        Onboard->>Repo: contract.tenant = tenant
    end

    Onboard->>Repo: status=ACTIVE
    Onboard->>DB: room.status = RENTED (nếu thuê phòng)
    Onboard->>Doc: autoGenerateDocument(contractId)
    Note right of Doc: DOCX bản chính thức (status=ACTIVE)

    Onboard-->>ActionCtrl: TenantContractResponse + tenantUsername
    ActionCtrl-->>FE: 200 OK
    FE-->>Manager: Hiển thị username khách (t{phone}) + mật khẩu mặc định 123456
```

## 8. So sánh với diagram cũ (đã sửa)

| Diagram cũ | Sửa thành |
|------------|-----------|
| `TenantContractService.createDraftContract()` | `TenantOnboardingServiceImpl.onboardTenant()` |
| `POST ...?draft=true` (query param) | `POST` body `{ draft: true }` |
| `PATCH` qua `TenantContractController` | `PATCH` qua `TenantContractActionController` |
| `NotificationService.notifyManager()` | **Chưa có** — đánh dấu planned |
| `pdfExtract.ts (OCR)` gọi như BE service | **FE client-side** — tách khỏi `OcrService` BE |
| Chỉ có tạo nháp + assign | Bổ sung deposit, PayOS, OTP, confirm, update draft |
| `TenantContract (DRAFT)` nhưng đã có tenant | `draft=true` → **không** tạo `User`/`Tenant` cho đến `confirm` |
| Thiếu `DRAFT → PENDING` | Xảy ra tại `createDepositPayment()` |
